# Security policy

Thank you for helping keep this project and its users safe.

## Reporting a vulnerability

Please email **<security-contact-email>** with a description of the issue. Include:

- A short summary of the problem
- Steps to reproduce, or a proof-of-concept
- The commit hash (or release tag) of the code you tested against
- The impact you believe it has

We acknowledge reports within **three business days** and aim to triage within **seven business days**. If you don't hear back, email again — it means the first message was lost, not ignored.

A PGP key for encrypted reports is available on request.

**Do not open a public GitHub issue for suspected security vulnerabilities.** Use the email address above. If you've already filed an issue, we will rewrite it to remove sensitive details and credit you in the resulting advisory unless you prefer to stay anonymous.
