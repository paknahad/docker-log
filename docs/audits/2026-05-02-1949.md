## Findings

### Selection quit can continue into streaming

`SelectionModel.Update` marks the model done for `q`, `ctrl+c`, and `enter`, but `runSelectionUI` always returns `selection.SelectedContainers()` without distinguishing quit from start. If a user selects containers and then presses `q` or `ctrl+c`, the CLI can receive those selected containers and proceed into the live log view instead of exiting.

This is a real workflow bug because quit keys should cancel the selection flow regardless of current selection state. A narrow fix should preserve selected containers only for the enter/start path and return no selection for quit.
